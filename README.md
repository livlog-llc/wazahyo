# 日本位置コード（和座標）設計書 v2

## 1. 目的

日本国内の位置を、**5文字の和文字コード**で表現します。将来的に以下の文化表現との連携を想定しています。

- 俳句
- 短歌
- 折句
- 作品タイトル

本方式では、**地域メッシュ体系**を利用して位置を機械的にエンコードします。

---

## 2. コード仕様

- コード長: **固定長5文字**
- 使用文字: **ひらがな + 漢字**

### 2.1 ひらがな（43文字）

```text
あいうえお
かきくけこ
さしすせそ
たちつてと
なにぬねの
はひふへほ
まみむめも
やゆよ
```

除外文字:

```text
を

```

除外理由:

- 語頭で使いにくい
- 折句と相性が悪い
- 入力ミスを防ぐ

## 5. 本方式で扱う範囲

日本国内の位置を以下の組合せで表現します。

- **1次メッシュ区分 × 2次〜9次メッシュ位置**

定義:

- **1次メッシュ区分**: 日本の1次メッシュを表す上位区分（**200パターン**）
- **2次〜9次メッシュ位置**: 各1次メッシュ内部の詳細位置

5文字コードはこの組合せ全体を表現します。

---

## 6. 必要コード数

### 6.1 1次メッシュ区分

```text
200
```

### 6.2 1区分あたりの詳細位置数

- 2次メッシュ: `8 × 8 = 64`
- 3次メッシュ: `10 × 10 = 100`
- 4〜9次メッシュ: `(2 × 2) を6回 = 4^6 = 4096`

### 6.3 詳細位置総数

```text
64 × 100 × 4096 = 26,214,400
```

### 6.4 日本全体の必要数

```text
26,214,400 × 200 = 5,242,880,000
```

## 7. 容量確認

したがって、本方式は容量的に成立します。

## 8. コード空間構造

5文字コードは整数として扱います。

- `n` の範囲: `0 〜 5,242,879,999`

`n` は以下の2要素で構成されます。

| 要素 | 範囲 |
| --- | --- |
| 1次メッシュ区分 | 0〜199 |
| 詳細位置 | 0〜26,214,399 |

## 9. エンコード方式

### ステップ1

緯度経度から以下を算出します。

- 1次メッシュ
- 2次〜9次メッシュ位置

### ステップ2

2次〜9次メッシュ位置を通し番号化します。

- 基数: `[8,8,10,10,4,4,4,4,4,4]`

```python
detail = 0
for value, base in zip(values, bases):
    detail = detail * base + value
```

- 結果: `detail = 0〜26,214,399`

### ステップ3

1次メッシュ区分を加算します。

```python
n = detail * 200 + mesh1_group
```

- `mesh1_group = 0〜199`

### ステップ4

整数 `n` を **88進5桁** に変換します。

---

## 10. デコード方式

### ステップ1

文字列を88進数として整数 `n` に変換します。

### ステップ2

1次メッシュ区分を取得します。

```python
mesh1_group = n % 200
```

### ステップ3

詳細位置番号を取得します。

```python
detail = n // 200
```

### ステップ4

`detail` を混合基数 `[8,8,10,10,4,4,4,4,4,4]` で展開します。

## 11. Python参考実装

### 11.1 文字セット

```python
HIRAGANA = "あいうえおかきくけこさしすせそたちつてとなにぬねのはひふへほまみむめもやゆよらりるれろわ"
KANJI = "春夏秋冬月雪風雨霧霞星空陽朝夕夜山川海島岬峰谷沢滝湖沼岸浜浦森林野原丘里村町都花草木竹石岩"
CHARS = HIRAGANA + KANJI
BASE = len(CHARS)
```

### 11.2 Base88変換

```python
def to_base88(n, length=5):
    s = []
    for _ in range(length):
        s.append(CHARS[n % BASE])
        n //= BASE
    return ''.join(reversed(s))

def from_base88(code):
    n = 0
    for c in code:
        n = n * BASE + CHARS.index(c)
    return n
```

### 11.3 詳細メッシュ番号化

```python
BASES = [8,8,10,10,4,4,4,4,4,4]

def encode_detail(parts):
    n = 0
    for v, b in zip(parts, BASES):
        n = n * b + v
    return n
```

### 11.4 復号

```python
def decode_detail(n):
    values = []
    for b in reversed(BASES):
        values.append(n % b)
        n //= b
    return list(reversed(values))
```

## 17. Java版ライブラリの使い方

このリポジトリには、README仕様を実装したJavaライブラリ（CLI付き）が含まれています。

### 17.1 ビルド

```bash
mvn -DskipTests package
```

生成物:

- `target/wazahyo-1.0.0.jar`

### 17.2 実行方法

#### デモ実行（引数なし）

```bash
java -jar target/wazahyo-1.0.0.jar
```

#### エンコード

```bash
java -jar target/wazahyo-1.0.0.jar encode <mesh1Group> <p0,p1,p2,p3,p4,p5,p6,p7,p8,p9>
```

例:

```bash
java -jar target/wazahyo-1.0.0.jar encode 42 1,2,3,4,1,0,2,3,1,2
```

#### デコード

```bash
java -jar target/wazahyo-1.0.0.jar decode <5-char-code>
```

例:

```bash
java -jar target/wazahyo-1.0.0.jar decode せ春里か湖
```

#### ヘルプ表示

```bash
java -jar target/wazahyo-1.0.0.jar help
```

### 17.3 引数の意味

- `mesh1Group`: `0〜199`
- `p0..p9`: 詳細メッシュ配列（混合基数）
  - `p0,p1`: `0〜7`
  - `p2,p3`: `0〜9`
  - `p4..p9`: `0〜3`
- `5-char-code`: ひらがな+漢字の5文字コード
